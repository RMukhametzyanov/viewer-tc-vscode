# Руководство пользователя: Test Case Viewer

## Что это

Расширение VS Code для работы с тест-кейсами в формате **Markdown** (`.md`):

- просмотр и редактирование в webview-панели;
- дерево тест-кейсов в рабочей области;
- фильтры по ключевым полям;
- вложения (`_attachment`);
- генерация статистики и отчетов (если команды доступны в вашей сборке).

## Установка из VSIX

### Через интерфейс VS Code

1. Откройте Extensions (`Ctrl+Shift+X`).
2. Нажмите `...` -> `Install from VSIX...`.
3. Выберите файл `*.vsix`.

### Через терминал

```bash
code --install-extension test-case-viewer-0.0.1.vsix
```

## Первый запуск

1. Откройте папку проекта в VS Code.
2. Откройте любой `.md` тест-кейс.
3. В Activity Bar откройте контейнер `Test Case Viewer`.
4. Используйте:
   - `Test Case Viewer` (webview редактор),
   - `Дерево объектов` (список/структура тест-кейсов).

## Верхняя панель действий ("шторка")

В webview сверху есть кнопка разворота панели действий:

- нажмите `Показать панель действий` - панель выпадет сверху с анимацией;
- нажмите повторно - панель свернется;
- состояние запоминается для webview.

В панели находятся кнопки:

- запуск прогона;
- статистика;
- настройки;
- генерация отчетов;
- обновление;
- показ/скрытие колонки статуса.

## Редактирование тест-кейса

- Изменения в полях сохраняются обратно в текущий markdown-файл.
- Для шагов доступны операции перемещения, добавления и удаления.
- Для комментариев и вложений доступны добавление/обновление/удаление через UI.

## Минимальный markdown-формат

```markdown
# Название тест-кейса

## Метаданные
| Поле | Значение |
|------|----------|
| **ID** | 123 |
| **Статус** | Draft |
| **Тип теста** | Manual |

## Описание (description)
Текст описания.

## Предусловия (preconditions)
Текст предусловий.

## Шаги тестирования
| Шаг | Действие | ОР | Статус |
|-----|----------|----|--------|
| 1 | Открыть страницу | Страница открылась | passed |
```

## Дерево объектов

Дерево строится по markdown-файлам workspace и показывает тест-кейсы.

Доступные действия:

- `Обновить`
- `Показать фильтры`
- `Сбросить фильтры`
- `Создать тест-кейс`
- `Создать папку`
- `Удалить`
- drag&drop для перемещения файлов и папок

## Частые проблемы

### Viewer пустой

- Проверьте, что активный файл - `.md`.
- Проверьте, что файл похож на тест-кейс (есть заголовок и шаги).

### Нет тест-кейса в дереве

- Обновите дерево командой `Обновить`.
- Убедитесь, что файл не отфильтрован активными фильтрами.

### Не видно колонку статуса шагов

- Нажмите кнопку `Показать статус` в верхней панели действий.
- Либо включите `testCaseViewer.showStatusColumn` в настройках VS Code.
