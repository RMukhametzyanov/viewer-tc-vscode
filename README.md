# Test Case Viewer

Расширение для VSCode, которое позволяет просматривать тест-кейсы в формате JSON в удобном читаемом виде.

## Возможности

- Автоматическое открытие preview панели при открытии JSON файла с тест-кейсом
- Красивое отображение всех полей тест-кейса
- Парсинг описания для извлечения информации о проекте, тест-плане и тест-сьюте
- Отображение шагов тестирования с описанием и ожидаемыми результатами
- Поддержка темной темы VSCode

## Установка

### Из исходников

1. Клонируйте репозиторий
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Скомпилируйте проект:
   ```bash
   npm run compile
   ```
4. Нажмите F5 в VSCode для запуска расширения в новом окне Extension Development Host
5. В новом окне откройте JSON файл с тест-кейсом

### Упаковка расширения

Для создания .vsix файла:

```bash
npm install -g vsce
vsce package
```

Затем установите через:
```
code --install-extension test-case-viewer-0.0.1.vsix
```

## Использование

1. Откройте JSON файл с тест-кейсом в VSCode (файл откроется как обычный JSON файл)
2. В боковой панели (Activity Bar) нажмите на иконку "Test Case Viewer"
3. В панели "Test Case Viewer" автоматически отобразится содержимое открытого тест-кейса
4. При переключении между тест-кейсами или изменении файла содержимое панели обновляется автоматически
5. Панель всегда видна и показывает содержимое активного тест-кейса

## Формат тест-кейса

Расширение ожидает JSON файлы со следующей структурой:

```json
{
    "id": "278512",
    "name": "Название тест-кейса",
    "description": "Проект: ...\nТест-план: ...\nТест-сьют: ...",
    "status": "Done",
    "testType": "automated",
    "priority": "2",
    "severity": "NORMAL",
    "testLayer": "E2E",
    "owner": "User.Name",
    "author": "User.Name",
    "reviewer": "User.Name",
    "steps": [
        {
            "id": "1",
            "name": "Шаг 1",
            "description": "Описание шага",
            "expectedResult": "Ожидаемый результат",
            "status": "skipped"
        }
    ]
}
```

## Разработка

```bash
# Компиляция
npm run compile

# Режим наблюдения
npm run watch
```
